# 開発PC 推奨スペック

Go2_deploy（Gazeboシミュレーション + Nav2ベースの自律スタック）を快適に動かすための
開発PCの目安。**購入・選定の根拠**として使う。数値は #44 の実測に基づく。

---

## 結論（先に要点）

- **CPUコア数が最重要**。物理シミュレーション・四足歩容・DWB経路追従・AMCL など、重い処理は
  ほぼ**CPU依存**。コアが多いほど余裕が出る。
- **GPU（できればNVIDIA）を強く推奨**。Gazebo/RViz の描画と3D LiDAR/カメラのセンサ生成を
  オフロードでき、GUIとRVizの同時表示やフリーズ回避に効く。**sim本体(upstream)はNVIDIA GPU前提**
  で設計されている（現状はiGPUで代替運用。NVIDIA自動対応は #46）。

---

## 推奨スペック表

| 項目 | 最低ライン | 推奨 | 備考 |
|------|-----------|------|------|
| **CPU** | 8コア | **12コア以上**（快適には16コア以上） | 現状12コア機で full stack(自律+可視化)はほぼ飽和。重い統合検証を日常的に回すなら余裕が欲しい |
| **GPU** | iGPU（Intel等、`/dev/dri`で描画可） | **NVIDIA（RTX相当）** | NVIDIAで描画・センサ生成をオフロード。sim upstreamの想定環境。iGPUでも動くが GUI+RViz同時は重い |
| **RAM** | 16GB | **32GB** | 現状 sim+自律スタックで実測 ~8GB使用。ビルドやツール併用も考慮 |
| **ストレージ** | 空き 50GB | **SSD 100GB以上** | dockerイメージ(数十GB) + ワークスペース + ビルド生成物 |
| **OS** | Ubuntu 22.04（ネイティブ） | Ubuntu ネイティブ | WSL2/デュアルブートは `docs/手順/` 参照。ネイティブが最も安定 |

---

## なぜこのスペックか（#44 実測の要点）

- **sim単体**：Gazebo物理+四足歩容+センサで **約3〜6コア**を消費（RTF=1.0を保つのに約8.7コア要求する場面も）。
- **full stack（自己位置推定+経路生成+経路追従+可視化）**：12コア機を**ほぼ使い切る**。
  - 特に **DWB経路追従コントローラがCPUを大量消費**（Nav2の性質）。
  - iGPU 1つで **Gazebo GUI と RViz を同時描画すると重く、フリーズ**することがある
    → 現状は **Gazeboヘッドレス（GUI窓オフ）+ RViz** の運用で回避（`dev-up.sh` の[4]でトグル）。
- **GPU（NVIDIA）があれば**：描画・センサ生成をGPUが担い、CPUと iGPU の取り合いが解消。
  GUI+RVizの同時表示やフリーズ回避、センサ高速化が期待できる。

## GPUについての補足

- 現状の docker 構成は iGPU（`/dev/dri`）前提。**NVIDIA GPU の自動検出・対応は #46** で対応予定
  （`nvidia-container-toolkit` + compose のGPU予約）。NVIDIA搭載PCを持つメンバーはそちらを参照。
- upstream の `go2_ros2_sim_py` は NVIDIA GPU 前提で書かれている（`docker/sim/compose.yaml` の
  コメント参照）。
