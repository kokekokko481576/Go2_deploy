# 経路生成 計画

## 1. 概要

Unitree Go2(顎LiDAR・背面アーム搭載)で船内の部材(防撓材等)まで移動し、溶接・検査作業を行うための経路生成系を構築する。

前提と分担:

- 全体マップは既知(図面ベース + 事前スキャン)
- 自己位置推定・経路追従は別計画。本計画の担当は「自己位置と目標部材位置を受け取り、Path を出す」まで
- 入力: `/map`(OccupancyGrid)、自己位置(map→base_link TF)、目標部材の座標と作業姿勢
- 出力: `nav_msgs/Path`(将来的にセグメント単位の「跨ぎスキル必要」タグ付き)

方針: 段階的に育てる。まず平地・障害物なしで部材へ一直線(Phase 1)、次にコストマップ + グリッド探索(Phase 2)、最後に高さコストと跨ぎ判断(Phase 3)。最初から本番と同じ I/F(Path 型、TF)で作り、中身だけ差し替えていく。

スコープ外: 部材の自動検出(当面は座標を手動指定)、経路追従(ローカル制御)、跨ぎ動作そのものの実行。

## 2. マイルストーン

| # | 内容 | 完了条件 |
|---|------|---------|
| M1 | 直線プランナ | 平地で指定部材の手前に正対して到達する Path を出せる |
| M2 | コストマップ + A* | 地図上の既知障害物を避ける経路を生成できる |
| M3 | ライブスキャン統合 | 地図にない障害物を obstacle layer で回避できる |
| M4 | 高さコスト | 標高マップから段差を3クラスにコスト化できる |
| M5 | 跨ぎ判断 | 迂回 vs 跨ぎをコスト比較で自動選択し、タグ付き経路を出せる |

M1〜M3 は Nav2 標準機能の構成が主。M4〜M5 が独自開発の本体。

## 3. 詳細

### 3.1 入出力とインタフェース定義

最初に固めるのはここ。ゴールは「部材の座標」ではなく「作業姿勢」で定義する: 部材に正対し、アームの作業リーチに入る距離 d だけ手前の点(d はアーム仕様から決定)。目標指定のメッセージ形式(部材の線分 + オフセット距離)を自己位置推定班・経路追従班と最初に合意しておく。Phase 1 では部材座標は手打ちで良い。

出力は `nav_msgs/Path`。Phase 3 以降は経路セグメントに属性(通常歩行 / 跨ぎスキル)を付ける必要があるため、拡張メッセージの形をこの段階で仮決めしておくと後の手戻りがない。

### 3.2 Phase 1: 直線プランナ(M1)

自己位置から目標作業姿勢まで直線補間した Path を吐くだけのノードを自作する。中身は数十行だが、目的は以下の確認:

- TF・Path・ゴール指定の I/F が経路追従側と噛み合うか
- 部材への正対精度(アーム作業の要求精度から逆算した許容値を決める)

Nav2 フル構成はまだ組まない。ただしトピック名・型は Nav2 と同じにしておき、Phase 2 で planner server に差し替えても追従側が気づかない状態を作る。

### 3.3 Phase 2: コストマップとグリッド探索(M2–M3)

Nav2 planner server を導入。プランナは NavFn(ダイクストラ/A*)または Smac 2D。

コストマップ構成(重ね順):

1. static layer: 既知マップ
2. obstacle layer: 顎LiDARのスキャンを2D投影して書き込み(マーク/レイトレースでクリア)— M3
3. inflation layer: 膨張半径 = 車体半幅 + マージン。アーム搭載時の外形で footprint を矩形定義

再計画は 1Hz または経路無効化時のみ。制御周期での回避はローカルコントローラ(経路追従班)の担当、という境界を守る。

### 3.4 Phase 3: 高さコストと跨ぎ判断(M4–M5)

顎LiDARの点群と自己位置から 2.5D 標高マップを構築(`grid_map` / `elevation_mapping` 系)。各セルの隣接段差を計算し、コストを3クラスに分類:

1. 段差 < 通常歩容限界(約10cm)→ 高さに応じた有限コスト
2. 通常限界〜学習スキル限界 → 跨ぎコスト + 「スキル必要」タグ
3. スキル限界超 → 致死

コスト単位は時間に統一する: 平地セル = 距離 ÷ 歩行速度、跨ぎセル = スキル実行時間 × 安全係数(+失敗リスクペナルティ)。これでダイクストラが迂回と跨ぎを同じ土俵で比較する。

防撓材は図面から位置・高さが既知なので、静的マップに事前注釈し、LiDAR実測は「高さ確認と位置ズレ補正」に使う。顎LiDARは前下方のみ視野があるため、標高マップは接近するまで埋まらない前提で設計する。

### 3.5 検証

- M1: 実機平地。部材正対精度・到達成功率を計測
- M2–M3: シミュレーション(Gazebo / Isaac Sim)で障害物配置を変えて経路品質を確認 → 実機
- M4–M5: 模擬防撓材(高さ可変)で、迂回/跨ぎの選択がコスト設定どおり切り替わることを確認
- 共通指標: 到達成功率、経路長、計画所要時間、ゴール姿勢誤差

### 3.6 リスク・未決事項

- 部材の指定方法の将来像(手動 → 図面連携 → LiDAR自動検出)
- アーム搭載による footprint・重心変化と膨張半径の再調整
- 跨ぎスキル(Isaac Lab)側の完成時期と M5 の依存関係 — M5 はスキルがダミー(その場停止)でもコスト判断のロジック自体は検証可能な構成にしておく
- タグ付き経路の受け渡し形式について経路追従班との調整
