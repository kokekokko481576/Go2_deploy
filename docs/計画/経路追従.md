# 経路追従 計画

## 1. 概要

経路生成が出す Path を受け取り、Go2 を実際に動かして部材前の作業姿勢まで到達させる。担当範囲は「Path + 自己位置 + コストマップを入力に、20Hzで `Twist (vx, vy, ω)` を決める」まで。歩容・バランス・足配置はUnitreeファーム(スポーツモード)の仕事で、本計画では触らない。

前提と分担:

- 出力I/F: `cmd_vel` → ドライバがスポーツモードのMove命令に変換
- Go2は横移動(vy)が使えるため、運動モデルは全方向(omni)として扱う
- 工場デモの衝突防止はNav2の回避と干渉するため、自律走行時はAPIでオフ(`ObstaclesAvoidSwitch`)。回避はこちらに一元化
- 防撓材の跨ぎスキル(Isaac Lab、低レベルモード)の中身は別開発。本計画が持つのは「切り替えの判断と手順」まで
- 溶接・検査作業中は静止。作業そのものはアーム側の担当

方針: まず手動でcmd_velパイプラインと安全機構を固め、Nav2コントローラサーバ(MPPIまたはDWB)で平地追従、最後に跨ぎスキルへの切り替えをビヘイビアツリーで統合する。

## 2. マイルストーン

| # | 内容 | 完了条件 |
|---|------|---------|
| M1 | cmd_velパイプライン | ROS2からGo2を速度指令で走らせられる。非常停止・タイムアウト停止が機能 |
| M2 | 平地の経路追従 | 直線Pathを追従し作業姿勢に到達(経路生成M1・自己位置M4と合流) |
| M3 | 局所回避 | 地図にない障害物をローカルコントローラだけで回避、リカバリ動作も動く |
| M4 | 最終アプローチ精度 | 部材正対のゴール姿勢誤差が要求値(アーム班と合意した値)以内 |
| M5 | 跨ぎスキル切り替え | タグ付き経路セグメントで 整列→スキル実行→Nav2復帰 の遷移が通る(スキルはダミー可) |

## 3. 詳細

### 3.1 入出力とインタフェース

入力: `nav_msgs/Path`(Phase 3以降はセグメント属性付き)、map→base_link TF、ローカルコストマップ、自己位置の共分散。出力: `geometry_msgs/Twist` 20Hz。ドライバ(unitree_ros2 / go2_ros2_sdk)がMove命令へ変換する。ローカルコストマップは自機周辺のローリングウィンドウ(例: 5m四方)で、顎LiDARのobstacle layer + inflationのみ。静的マップは持たない。

### 3.2 M1: cmd_velパイプラインと安全機構

Nav2より先に、速度指令系と安全を固める:

1. ドライバ導入、テレオペ(ジョイパッド)でcmd_vel走行確認
2. 工場衝突防止のオフをAPIから実行し、起動シーケンスに組み込む
3. 安全機構: 速度・加速度の上限クランプ、cmd_velが途絶えたら停止するウォッチドッグ(0.5s)、無線非常停止
4. 指令速度に対する実応答(遅れ・立ち上がり)を計測 — コントローラの前方シミュレーションのモデル精度に直結

### 3.3 M2: コントローラ選定と平地追従

Nav2 controller server を導入。候補はMPPI(第一候補: omni対応が素直、障害物コストの扱いが柔軟)とDWB(枯れている、パラメータが直感的)。共通設定:

- 運動モデル: omni(vyをサンプリング空間に含める)。狭所での平行移動が効く
- footprint: アーム搭載時の外形矩形(経路生成側と同一定義を共有)
- 制御周期20Hz、前方シミュレーション1.5s程度
- goal checker: 位置・向きの許容値をM4の要求から逆算して設定

M2完了 = 経路生成M1・自己位置推定M4との三者合流点。ここで全体パイプラインが初めて通る。

### 3.4 M3: 局所回避とリカバリ

地図にない障害物(床上の資材等)をローカルコストマップだけで回避できることを確認。合わせて:

- リカバリ動作(その場回転でのコストマップクリア、後退)をBTに設定
- 自己位置の共分散が大きいときに速度を落とすスケーリングを実装(自己位置推定計画3.1のI/Fを使用)
- 回避不能(通路閉塞)時にグローバル再計画を要求する遷移の確認

### 3.5 M4: 最終アプローチ

到達を二段に分ける: 通常追従でゴール手前(例: 0.5m)まで来たら低速の最終アプローチモードに入り、vyと超信地旋回で部材に正対する。ゴール姿勢誤差の要求値はアーム班と合意して数値で固定する(グローバル自己位置は10cm精度なので、それより厳しい要求はアーム側の部材相対位置合わせで吸収する、という自己位置推定計画の分担を守る)。

### 3.6 M5: 跨ぎスキルへの切り替え

ビヘイビアツリーに以下の遷移を実装:

1. 経路の「スキル必要」タグ区間に接近したら通常追従を停止
2. 整列: 防撓材に対して垂直・所定距離に位置合わせ(標高マップまたは事前注釈から防撓材の線分を取得)
3. スポーツモード → 低レベルモードへ切り替え、学習ポリシーに制御を渡す(この間cmd_velは無効。自己位置はIMU保持 — 自己位置推定計画3.2)
4. 着地・完了判定後、スポーツモードに復帰しNav2再開

スキル本体が未完成でも、ダミー(整列して静止→復帰)で遷移ロジック自体はM5として検証する。モード切り替え中の安全(転倒検知、中断時の復帰手順)はスキル班と共同で設計。

### 3.7 検証

- 指標: 経路横断誤差(追従中)、ゴール姿勢誤差、障害物回避成功率、リカバリ発動頻度、モード切り替えの所要時間と成功率
- 環境: シミュレーション(Gazebo / Isaac Sim)でM2–M3を先行 → 実機平地 → 模擬防撓材
- M1の安全機構は毎マイルストーンで回帰確認(ウォッチドッグ・非常停止)

### 3.8 リスク・未決事項

- スポーツモードの速度応答の遅れが大きい場合、前方シミュレーションとの乖離で追従が振動する(M1の応答計測で早期に把握)
- アーム+溶接機材の重量・重心変化で許容加速度が変わる(実機計測後にクランプ値を再調整)
- 低レベルモード⇔スポーツモードの切り替え挙動(所要時間、切り替え時の姿勢制約)が未確認 — M5前に単体で検証
- ゴール姿勢誤差の要求値がアーム班と未合意(M4の前提。早期に数値を固める)
