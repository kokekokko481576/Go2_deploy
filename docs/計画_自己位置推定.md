# 自己位置推定 計画

## 1. 概要

既知マップ上で Go2 の姿勢(map→base_link)を連続推定し、経路生成・経路追従に供給する。

前提と分担:

- マップは既知(図面ベース + 事前スキャンで補正)。SLAM(地図構築)は本計画のスコープ外だが、事前スキャンの手段としてのみ使う
- センサ: 顎の3D LiDAR、本体IMU、Unitree脚オドメトリ(スポーツモード状態から取得)
- 出力: map→odom TF(補正分)+ 推定姿勢と共分散。odom→base_link はオドメトリ側が高周波で出す、という標準の二段構成
- 利用側の要求精度で設計を割り切る: 経路生成・追従に必要なのは数cm〜10cm。溶接に必要なmm精度の最終位置決めは、グローバル自己位置ではなく部材相対の位置合わせ(アーム側センシング)で出す。この分担を最初に確定させる

方針: まず脚オドメトリ+IMUの融合で odom を安定させ(ここが脚ロボの肝、車輪より滑る)、その上に 2D AMCL を載せる。3D化(点群 vs 点群地図のマッチング)は精度不足が確認されてから。

## 2. マイルストーン

| # | 内容 | 完了条件 |
|---|------|---------|
| M1 | オドメトリ整備 | 脚オドメトリ+IMUをEKF融合し、平地10m歩行でドリフトを定量化 |
| M2 | AMCL稼働 | 既知マップ・平地で連続推定、誤差10cm以内を維持 |
| M3 | 精度・頑健性評価 | 初期位置指定からの収束、ロスト復帰、対称空間での挙動を確認 |
| M4 | 経路生成との結合 | TF経由でPhase 1直線プランナが動く(経路生成M1と合流) |
| M5 | 標高マップ用の姿勢供給 | z・ロール・ピッチ込みの姿勢で標高マップが歪まないことを確認 |

## 3. 詳細

### 3.1 TF構成とインタフェース

`map → odom → base_link` の標準構成。odom→base_link はオドメトリノードが50Hz以上で発行、map→odom はAMCLが低周波で補正する。base_link→LiDAR・アーム基部の静的TFは実測でキャリブレーションし、URDFに固定。共分散(`/amcl_pose`)も出力し、経路追従側が「自信がないときは減速する」判断に使えるようにする。

### 3.2 M1: オドメトリ(脚+IMU融合)

脚オドメトリ(関節エンコーダの順運動学 + 接地判定 + IMU融合)は Unitree ファームウェアが本体内で計算済みで、状態メッセージ(`rt/sportmodestate` 等)に推定位置・速度として乗ってくる。順運動学の自作は不要。M1の作業は:

1. ROS2ドライバ(unitree_ros2 / go2_ros2_sdk)を導入し `/odom` とTFを取得
2. IMUと合わせて `robot_localization` のEKFに通す(設定記述のみ)
3. 平地10m歩行でドリフト率(m/m、deg/m)を実測

脚オドメトリは着地時の微小な滑り・沈みで一歩ごとに誤差が積むため、実測値がAMCLの予測ステップのノイズパラメータ(alpha)設定の根拠になる。

例外は低レベルモード(関節直接制御)で動かす区間。Isaac Lab学習スキルの実行中はスポーツモードを離れるため、本体のオドメトリが使えない・信用できない可能性がある。対応方針(優先順):

1. 跨ぎは数秒なので、スキル実行中はIMUのみで姿勢を保持し、着地後にAMCLで補正(推奨・まずこれ)
2. 精度不足なら、URDFベースの運動学ライブラリ(Pinocchio / KDL)+ 脚ロボ用状態推定器のオープンソース実装を組み合わせて自前推定 — フルスクラッチはしない

どちらにするかの判断はM3の評価と跨ぎスキル側の進捗を見て決める。

### 3.3 M2: AMCL

顎LiDARの点群を高さ帯(例: 床上10〜50cm)でスライスして2D LaserScanに変換し、AMCLに入れる。設定の要点:

- 初期位置は手動指定(RViz)。船内は対称・反復構造が多く、大域的初期化は当てにしない
- パーティクル数・尤度場σ・motion model のalphaをM1の実測ドリフトから設定
- マップは図面由来のため実構造とズレがある可能性 → 事前スキャン(slam_toolbox)で作った地図と図面地図の両方で精度比較し、良い方を採用

### 3.4 M3: 頑健性評価

評価項目: (1) 初期位置を1m/30°ずらした状態からの収束時間、(2) 長い一様な通路(特徴欠乏)でのドリフト、(3) 同型区画が並ぶ対称環境での誤収束の有無、(4) 人・障害物がスキャンの一部を遮った場合の劣化。鋼板の反射によるLiDAR異常値(マルチパス)の頻度もここで確認し、必要なら距離・強度フィルタを入れる。

### 3.5 M4–M5: 下流との結合

M4は経路生成計画のM1と同一イベント(直線プランナがTFで動く)。M5では標高マップ構築に6自由度姿勢が必要になるため、EKF出力のz・ロール・ピッチの品質を確認する。2D AMCLはx, y, yawしか補正しないので、z方向の累積誤差が標高マップを歪ませる場合は、床面検出でzをリセットする等の対策を検討。

### 3.6 検証

- 真値: 区画内の既知点(墨出しマーク)への手動突き合わせ + トータルステーションがあれば併用
- 指標: 定常誤差(x, y, yaw)、収束時間、ロスト頻度、CPU負荷
- 環境: 実験室平地 → 模擬防撓材あり床 → 実船区画(またはモックアップ)

### 3.7 リスク・未決事項

- 図面マップと実構造の乖離度が未知(まず事前スキャンとの差分を測る)
- 低レベルモード中のオドメトリ欠落(3.2の対応方針1で足りるかは跨ぎスキルの動作の激しさ次第。経路追従計画と共同で扱う)
- 溶接時のヒューム・スパッタがLiDARに与える影響(作業中は移動しない前提でどこまで逃げられるか)
- 部材相対の最終位置合わせをどちらの計画が持つか(本計画は候補提示のみ: アーム側とする案を推奨)
