# Windows + Ubuntu 26.04 デュアルブート構築手順 (Issue #40)

既に Windows が入った PC に Ubuntu 26.04 LTS を追加し、起動時に OS を選べるように
する手順。インストールメディア（USB）は用意済みの前提。**26.04 のインストーラは
「Windows の隣にインストール」を選ぶだけで自動でパーティションを切ってくれるので
かなり楽**。ここではその自動パスと、細かく制御したい人向けの手動パーティション
推奨の両方を書く。

> このプロジェクトの Docker 環境は **ストレージ 30〜40GB・RAM 16GB** を要する
> （`docker/README.md` 系の要件）。パーティション設計はこれを見込んで決める。

---

## ⚠️ 始める前に（データ保護・最重要）

- **バックアップ**: パーティション操作は失敗するとデータを失う。重要データは外部へ退避。
- **BitLocker の確認/一時停止**: Windows のディスクが BitLocker 暗号化されていると、
  パーティション変更後に Windows が回復キーを要求する。Windows の「BitLocker の管理」で
  **一時停止**するか、回復キーを控えておく（Microsoft アカウントにも保存される）。
- **高速スタートアップ(Fast Startup)を無効化**: 有効だと Windows のファイルシステムが
  「休止状態」で保持され、Linux 側からのアクセスや再起動でトラブルになる。
  Windows: コントロールパネル →「電源オプション」→「電源ボタンの動作を選択する」→
  「現在利用可能でない設定を変更します」→「高速スタートアップを有効にする」の
  **チェックを外す**。
- **UEFI か Legacy(BIOS) か**: 今の Windows がどちらで入っているかに Ubuntu も合わせる
  （混在すると起動メニューに出ない）。近年の機はほぼ **UEFI**。本手順は UEFI 前提で書く。

---

## 手順1: Windows 側で空き領域を作る

インストーラの自動縮小でもよいが、Windows のツールで先に空けておくと安全・確実。

1. Windows で「ディスクの管理」を開く（スタート右クリック →「ディスクの管理」）。
2. Cドライブ（通常 `C:`）を右クリック →「ボリュームの縮小」。
3. Ubuntu 用に空ける量を入力。**最低 120GB、できれば 200GB 以上**を推奨
   （後述のパーティション設計と、Docker イメージ/ビルドの余裕分）。
4. 縮小後、その領域は「未割り当て」のまま残す（フォーマットしない）。

---

## 手順2: UEFI/BIOS 設定

1. PC を再起動し、起動直後にメーカーキー（`F2`/`F10`/`F12`/`Del` など）で UEFI 設定へ。
2. **Secure Boot**: 26.04 は署名済みなので **有効のままでインストール可能**。
   iGPU 運用ならこのままで問題ない（NVIDIA の追加ドライバを後で入れるなら、その時に
   MOK 登録を求められる／一時的に無効化する。本プロジェクトの iGPU 用途では不要）。
3. **起動順**: USB を最優先に。または起動時のワンタイムブートメニュー（`F12` 等）で
   USB を選ぶ。
4. 設定を保存して、Ubuntu の USB を挿したまま再起動。

---

## 手順3: インストーラ起動 〜 種類の選択

1. USB から起動し「Try or Install Ubuntu」を選ぶ。
2. 言語・キーボードを選択。ネット接続（有線推奨）で更新を取り込む。
3. インストール種類の画面で、以下のどちらかを選ぶ。

### 【推奨・楽な方】Windows の隣にインストール（自動）

- **「Install Ubuntu alongside Windows Boot Manager」** を選ぶと、手順1で空けた
  未割り当て領域（または既存 Windows を自動縮小して）に、インストーラが
  ESP 流用・root・swap を自動で切ってくれる。スライダで Ubuntu に割く容量を調整できる。
- 迷ったらこれでよい。**26.04 が「楽」なのはこの自動配置が賢いから**。
- 割り当て容量だけは、下の推奨（root 実質 120GB 以上）を満たすよう多めに。

### 【細かく制御したい人向け】手動（"Something else" / 手動パーティション）

「それ以外」「手動パーティション」を選び、手順1の未割り当て領域に下表のとおり作る。

| マウントポイント | 種類 | サイズの推奨 | 備考 |
|---|---|---|---|
| `/boot/efi`（ESP） | EFI System Partition (fat32) | **既存 Windows の ESP を流用**（新規作成なら 512MB〜1GB） | UEFI では 1 台に 1 つでよい。**フォーマットしない**で「利用」だけ指定 |
| `/`（root） | ext4 | **120GB 以上**（最低 100GB。可能なら 200GB+） | Docker イメージ＋colcon ビルドで 30〜40GB＋作業余裕。ここをケチると後で詰まる |
| swap | swap 領域 | RAM 16GB なら **8〜16GB**（休止状態を使うなら RAM 以上） | 省略して root 上の swapfile 任せでも可（26.04 既定） |
| `/home` | ext4 | 残り（分けるなら 50GB 以上） | **任意**。分けると OS 再インストール時にデータを残しやすい。1 台運用なら分けなくても可 |

- **ブートローダ(GRUB)のインストール先**: ディスク全体（例 `/dev/nvme0n1`、パーティション
  ではなくディスク本体）を指定。既存の Windows Boot Manager と共存する GRUB が入る。
- シンプルにしたいなら: **ESP 流用 + `/`(ext4, 大きめ) + swap** の 3 つで十分。
  `/home` 分割は好みで。

---

## 手順4: インストール実行 〜 初回起動

1. ユーザ名・パスワード・タイムゾーンを設定してインストール実行。
2. 完了後 USB を抜いて再起動すると **GRUB メニュー**が出る。
   - `Ubuntu` … Ubuntu で起動
   - `Windows Boot Manager` … Windows で起動
   - 既定は Ubuntu。数秒で自動選択される。
3. GRUB に Windows が出ない場合は、Ubuntu 起動後に次で再検出:

   ```bash
   sudo apt-get update && sudo apt-get install -y os-prober
   sudo os-prober            # Windows が見つかるか確認
   sudo update-grub          # GRUB メニューに反映
   ```

---

## 手順5: 初回のお約束（時計ずれ対策など）

- **時計がずれる問題**: Windows はハード時計をローカル時刻、Linux は UTC として扱うため、
  デュアルブートで時刻がずれることがある。Ubuntu 側を Windows に合わせるなら:

  ```bash
  timedatectl set-local-rtc 1 --adjust-system-clock
  ```

- **更新**: `sudo apt-get update && sudo apt-get upgrade -y`
- **GPU**: iGPU は Ubuntu 標準の mesa で概ねそのまま動く（本プロジェクトの用途では
  追加ドライバ不要）。「追加ドライバ」を入れるのは NVIDIA 機の話（#33）。

## トラブルシュート

- **再起動したら Windows が直接起動して GRUB が出ない**: UEFI 設定のブート順で
  `ubuntu` を Windows Boot Manager より上にする。または `sudo update-grub` を実行。
- **Windows が回復キーを要求する**: BitLocker が有効だった → 控えた回復キー（または
  Microsoft アカウントの https://account.microsoft.com/devices/recoverykey ）で解除。
  始める前に一時停止しておけば回避できる。
- **パーティション縮小で「これ以上縮小できない」**: Windows の移動不可ファイル
  （ページファイル・休止ファイル・システム保護）が邪魔している。高速スタートアップ・
  休止を無効化し、ページファイルを一時無効・デフラグしてから再挑戦。
- **USB から起動できない**: Secure Boot は 26.04 なら有効のままで可。それでも駄目なら
  一時的に無効化、および UEFI/Legacy がメディアと一致しているか確認。
